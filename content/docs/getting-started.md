Stash is a browser extension that lets you share multiple tabs with a single URL. Perfect for sharing research, projects, or collections of websites with colleagues, friends, or yourself across devices.

## Installation

### Download from GitHub

1. Go to [stash/releases](https://github.com/espetro/stash/releases) and download the latest version for your browser:
   - **Chrome**: Download `stash-extension-*-chrome.zip`
   - **Firefox**: Download `stash-extension-*-firefox.zip`

### Install in Chrome
1. Unzip the downloaded file
2. Navigate to `chrome://extensions`
3. Enable **Developer Mode** (toggle in the top right)
4. Click **Load unpacked**
5. Select the unzipped folder and click **Select**

### Install in Firefox
1. Navigate to `about:debugging`
2. Click **This Firefox** (on the left sidebar)
3. Click **Load Temporary Add-on**
4. Select the downloaded `.zip` file

_We're working on getting Stash listed in the Chrome Web Store and Firefox Add-ons — in the meantime, install directly from the releases page above._

### No extension? Use the web app

Prefer not to install? Try [Stash in your browser](/s/new) — no extension or account required.

## Quick Start: Share Your First Tabs

Here's how to share tabs in 3 simple steps:

### 1. Select Tabs
Open the tabs you want to share. Hold **Cmd+Click** (Mac) or **Ctrl+Click** (Windows/Linux) to select multiple tabs, or **Shift+Click** to select a consecutive range.

### 2. Right-Click
Right-click on any selected tab and choose **"Share selected tabs…"** from the context menu.

### 3. Get Link
A share link will be copied to your clipboard automatically. You can now paste this link anywhere—email, messaging apps, or save it for later.

## What Happens Next?

When someone opens your share link, they'll see a clean page with all your tabs listed. Each tab shows the title and can be opened with a single click. Links expire automatically based on your settings (default is never).

## Privacy Note

Stash works entirely in your browser. No data is sent to any server—your tabs are encoded directly into the share link itself. You can learn more in our [privacy policy](/privacy).
